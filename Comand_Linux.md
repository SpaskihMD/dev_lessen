# 🐧 Основы работы в Linux (шпаргалка)

## 🔍 Навигация и информация о системе

| Команда | Описание | Пример |
|---------|----------|--------|
| `whoami` | Показывает текущего пользователя | `whoami` → `alex` |

## 📄 Работа с файлами (просмотр, создание, удаление)

### Просмотр содержимого
```bash
cat filename          # прочитать файл
head -n 10 filename   # первые 10 строк (n — любое число)
tail -n 10 filename   # последние 10 строк
less filename         # постраничный просмотр (выход — q, поиск — /слово)
```

### Создание файлов
```bash
cat > 123.txt         # создать файл и записать текст (Ctrl+D — сохранить и выйти)
touch file.txt        # создать пустой файл или обновить дату
touch file{1,2,3}.txt # создать несколько файлов сразу
```

### Удаление
```bash
rm file.txt           # удалить файл
rmdir empty_folder    # удалить пустую папку
rm -r folder          # удалить папку со всем содержимым
rm -rf folder         # удалить без подтверждения (⚠️ осторожно!)
```

## 🔄 Перенаправление вывода (`>`, `>>`, `|`)

| Символ | Назначение | Пример |
|--------|------------|--------|
| `>` | Создать/перезаписать файл выводом команды | `ls > files.txt` |
| `>>` | Добавить вывод в конец файла | `echo "новая строка" >> files.txt` |
| `\|` (pipe) | Передать вывод одной команды на вход другой | `ls \| grep ".txt"` |

**Примеры:**
```bash
ls --help > help.txt          # записать справку в файл
cat 123.txt > 111.txt         # скопировать содержимое (с заменой)
cat 123.txt >> 111.txt        # добавить в конец
ls -l | head -10              # первые 10 строк из ls -l
```

## 📋 Команда `ls` (просмотр содержимого)

```bash
ls              # список файлов в текущей папке
ls -l           # подробная информация (права, владелец, размер, дата)
ls -a           # показать скрытые файлы (начинаются с .)
ls -t           # сортировка по времени изменения
ls *            # показать папки и их содержимое
ls -l | head    # первые 10 строк вывода
```

## 🔎 Поиск с `grep` (очень мощная команда)

### Базовый синтаксис
```bash
grep "слово" filename.txt          # поиск в одном файле
grep -r "функция" ./project/       # рекурсивный поиск по всем файлам в папке
grep -i "hello" file.txt           # без учета регистра
grep -w "cat" file.txt             # только целое слово
```

### Полезные флаги

| Флаг | Назначение | Пример |
|------|------------|--------|
| `-i` | Игнорировать регистр | `grep -i "error" log.txt` |
| `-w` | Только целые слова | `grep -w "main" script.py` |
| `-v` | Исключить строки с совпадением | `grep -v "debug" log.txt` |
| `-n` | Показать номера строк | `grep -n "TODO" code.py` |
| `-c` | Показать количество совпадений | `grep -c "apple" fruit.txt` |
| `-r` | Рекурсивный поиск (во всех подпапках) | `grep -r "config" ./` |
| `-l` | Только имена файлов с совпадением | `grep -l "error" *.log` |
| `-A 2` | +2 строки после совпадения | `grep -A 2 "Exception" log.txt` |
| `-B 3` | +3 строки до совпадения | `grep -B 3 "warning" log.txt` |
| `--color` | Подсветка совпадений | `grep --color "error" log.txt` |

### Работа с архивами
```bash
unzip -p archive.zip | grep -i "искомое"   # поиск в ZIP-архиве
unzip -p *.zip | grep -i "что_ищешь"      # поиск по всем ZIP-архивам в папке
```

## 📦 Копирование и перемещение

### `cp` — копирование
```bash
cp source.txt dest.txt                # скопировать файл
cp source.txt /home/user/backup/      # скопировать в другую папку
cp -r source_folder/ dest_folder/     # рекурсивно (всю папку)
cp -u source.txt dest.txt             # копировать, только если source новее или dest отсутствует
cp -ru source/ dest/                  # рекурсивно + только новые/обновленные
```

**Флаги `cp`**:
- `-r` — рекурсивное копирование (для папок)
- `-u` — update (копировать только новое или отсутствующее)
- `-p` — сохранить права, владельца, время
- `-v` — подробный вывод (verbose)

|-----------|---------|
| `*` | любой набор символов (wildcard) |
| `./*` | все файлы в текущей папке |
| `*.txt` | все файлы с расширением .txt |
| `-r` | рекурсивно (обрабатывать вложенные папки) |
| `-f` | force (игнорировать предупреждения) |
| `-v` | подробный вывод (verbose) |

## 💡 Комментарии к ключевым моментам

- **`cp -u`** — отличная альтернатива `rsync` для простых случаев, копирует только изменённые файлы
- **`grep -r`** — настоящий супергерой для поиска по проекту
- **`>` и `>>`** — основа для логирования и сохранения результатов
- **`|` (pipe)** — позволяет строить цепочки команд, где результат одной идёт на вход другой
- **`man`** — лучший друг, если забыл синтаксис

# 📦 Команда `mv` — перемещение и переименование файлов

## 🎯 Основное назначение

`mv` (от англ. **move**) выполняет две функции:
1. **Перемещает** файлы и папки из одного места в другое
2. **Переименовывает** файлы и папки

---

## 📝 Базовый синтаксис

```bash
mv [опции] источник назначение
```

---

## 🔄 Переименование (работает как "rename")

### Переименовать файл
```bash
mv oldname.txt newname.txt
```

### Переименовать папку
```bash
mv old_folder/ new_folder/
```

**Важно:** При переименовании папки с содержимым — всё содержимое остаётся нетронутым, меняется только имя папки.

---

## 📂 Перемещение

### Переместить файл в другую папку
```bash
mv file.txt /home/user/documents/
```

### Переместить несколько файлов в папку
```bash
mv file1.txt file2.txt file3.txt /target/folder/
```

### Переместить папку в другую папку
```bash
mv source_folder/ /home/user/destination/
```

### Переместить всё содержимое папки
```bash
mv source_folder/* destination_folder/
```

---

## 🚩 Полезные флаги

| Флаг | Описание | Пример |
|------|----------|--------|
| `-i` | **interactive** — запрашивает подтверждение перед перезаписью | `mv -i file.txt target/` |
| `-u` | **update** — перемещает только если источник новее или целевой отсутствует | `mv -u source.txt target/` |
| `-v` | **verbose** — показывает подробности выполнения | `mv -v *.txt target/` |
| `-n` | **no-clobber** — не перезаписывать существующие файлы | `mv -n file.txt target/` |
| `-b` | **backup** — создаёт резервную копию перед перезаписью | `mv -b file.txt target/` |

---

## 💡 Комбинации и примеры

### 1. Переместить с подтверждением
```bash
mv -i document.txt /backup/
# Если файл существует, спросит: overwrite? (y/n)
```

### 2. Переместить только новые файлы
```bash
mv -u *.log /var/logs/archive/
# Переместит только те .log файлы, которых нет в archive или они новее
```

### 3. Переместить и показать что происходит
```bash
mv -v *.jpg images/
# Вывод: 'photo1.jpg' -> 'images/photo1.jpg'
```

### 4. Переместить все файлы определённого типа
```bash
mv *.pdf ~/documents/pdfs/
```

### 5. Переместить папку и переименовать одновременно
```bash
mv project/ /backup/old_project/
# Перемещает папку project в /backup/ и переименовывает в old_project
```

---

## ⚠️ Важные нюансы

### 🔴 Если назначение — папка
```bash
mv file.txt existing_folder/   # переместит ВНУТРЬ папки
```

### 🟡 Если назначение — не существующая папка
```bash
mv file.txt new_folder/        # если new_folder не существует — ошибка
mv file.txt new_folder         # если new_folder не существует — ПЕРЕИМЕНУЕТ файл в "new_folder"
```

### 🟢 Правило:
- Если **назначение существует и это папка** → файл перемещается внутрь
- Если **назначение не существует** → файл переименовывается
- Если **назначение существует и это файл** → файл перезаписывается (если нет `-i`)

---

## 🆚 Отличие `mv` от `cp`

| `mv` | `cp` |
|------|-----|
| Перемещает (стирает оригинал) | Копирует (сохраняет оригинал) |
| Работает мгновенно (просто меняет путь) | Копирует содержимое (занимает время) |
| Можно использовать для переименования | Нельзя переименовать без копирования |

---

## 🧠 Примеры для практики

```bash
# 1. Переименовать файл
mv report.txt final_report.txt

# 2. Переместить файл в другую папку
mv image.png ~/Pictures/

# 3. Переместить все .txt файлы в папку backup
mv *.txt backup/

# 4. Переместить с подтверждением и подробным выводом
mv -iv *.log /var/logs/

# 5. Переместить папку и всё её содержимое
mv old_project/ /home/user/archived_projects/

# 6. Переместить, но не перезаписывать существующие файлы
mv -n file.txt destination/

# 7. Создать резервную копию при перезаписи
mv -b config.conf /etc/config.conf
# Создаст /etc/config.conf~ (старую версию)
```

## 🎯 Короткая шпаргалка

| Задача | Команда |
|--------|---------|
| Переименовать файл | `mv старое_имя новое_имя` |
| Переименовать папку | `mv старая_папка новая_папка` |
| Переместить файл в папку | `mv файл папка/` |
| Переместить несколько файлов | `mv file1 file2 file3 папка/` |
| Переместить папку | `mv папка новое_место/` |
| С подтверждением | `mv -i источник назначение` |
| С подробным выводом | `mv -v источник назначение` |
| Только если новее | `mv -u источник назначение` |

---

## 💡 Совет

Используй `mv -iv` в комбинации:
- `-i` защитит от случайной перезаписи
- `-v` покажет, что именно произошло

Это особенно полезно при работе с важными файлами! 🚀
