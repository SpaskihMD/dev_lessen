## 📄 Работа с файлами (просмотр, создание, удаление)

```bash
cat filename          # прочитать файл
```

```bash

touch file{1,2,3}.txt # создать несколько файлов сразу
```

### Удаление
```bash
rm file.txt           # удалить файл
rmdir empty_folder    # удалить пустую папку
rm -r folder          # удалить папку со всем содержимым
rm -rf folder         # удалить без подтверждения (⚠️ осторожно!)
```

## 🔄 Перенаправление вывода (`>`, `>>`, `|`)

| Символ | Назначение | Пример |
|--------|------------|--------|
| `>` | Создать/перезаписать файл выводом команды | `ls > files.txt` |
| `>>` | Добавить вывод в конец файла | `echo "новая строка" >> files.txt` |
| `\|` (pipe) | Передать вывод одной команды на вход другой | `ls \| grep ".txt"` |

**Примеры:**
```bash
ls --help > help.txt          # записать справку в файл
cat 123.txt > 111.txt         # скопировать содержимое (с заменой)
cat 123.txt >> 111.txt        # добавить в конец
ls -l | head -10              # первые 10 строк из ls -l
```

```bash
ls              # список файлов в текущей папке
```

## 🔎 Поиск с `grep` (очень мощная команда)

### Базовый синтаксис
```bash
grep "слово" filename.txt          # поиск в одном файле
grep -r "функция" ./project/       # рекурсивный поиск по всем файлам в папке
grep -i "hello" file.txt           # без учета регистра
grep -w "cat" file.txt             # только целое слово
```

### Полезные флаги

| Флаг | Назначение | Пример |
|------|------------|--------|
| `-i` | Игнорировать регистр | `grep -i "error" log.txt` |
| `-w` | Только целые слова | `grep -w "main" script.py` |
| `-v` | Исключить строки с совпадением | `grep -v "debug" log.txt` |
| `-n` | Показать номера строк | `grep -n "TODO" code.py` |
| `-c` | Показать количество совпадений | `grep -c "apple" fruit.txt` |
| `-r` | Рекурсивный поиск (во всех подпапках) | `grep -r "config" ./` |
| `-l` | Только имена файлов с совпадением | `grep -l "error" *.log` |
| `-A 2` | +2 строки после совпадения | `grep -A 2 "Exception" log.txt` |
| `-B 3` | +3 строки до совпадения | `grep -B 3 "warning" log.txt` |
| `--color` | Подсветка совпадений | `grep --color "error" log.txt` |

### Работа с архивами
```bash
unzip -p archive.zip | grep -i "искомое"   # поиск в ZIP-архиве
unzip -p *.zip | grep -i "что_ищешь"      # поиск по всем ZIP-архивам в папке
```

## 📦 Копирование и перемещение

### `cp` — копирование
```bash
cp source.txt dest.txt                # скопировать файл
cp source.txt /home/user/backup/      # скопировать в другую папку
cp -r source_folder/ dest_folder/     # рекурсивно (всю папку)
cp -u source.txt dest.txt             # копировать, только если source новее или dest отсутствует
cp -ru source/ dest/                  # рекурсивно + только новые/обновленные
```

**Флаги `cp`**:
- `-r` — рекурсивное копирование (для папок)
- `-u` — update (копировать только новое или отсутствующее)
- `-p` — сохранить права, владельца, время
- `-v` — подробный вывод (verbose)

# 📦 Команда `mv` — перемещение и переименование файлов

## 🎯 Основное назначение

`mv` (от англ. **move**) выполняет две функции:
1. **Перемещает** файлы и папки из одного места в другое
2. **Переименовывает** файлы и папки

---

## 📝 Базовый синтаксис

```bash
mv [опции] источник назначение
```

---

## 🔄 Переименование (работает как "rename")

### Переименовать файл
```bash
mv oldname.txt newname.txt
```

### Переименовать папку
```bash
mv old_folder/ new_folder/
```

**Важно:** При переименовании папки с содержимым — всё содержимое остаётся нетронутым, меняется только имя папки.

---

## 📂 Перемещение

### Переместить файл в другую папку
```bash
mv file.txt /home/user/documents/
```

### Переместить несколько файлов в папку
```bash
mv file1.txt file2.txt file3.txt /target/folder/
```

### Переместить папку в другую папку
```bash
mv source_folder/ /home/user/destination/
```

### Переместить всё содержимое папки
```bash
mv source_folder/* destination_folder/
```

---

## 🚩 Полезные флаги

| Флаг | Описание | Пример |
|------|----------|--------|
| `-i` | **interactive** — запрашивает подтверждение перед перезаписью | `mv -i file.txt target/` |
| `-u` | **update** — перемещает только если источник новее или целевой отсутствует | `mv -u source.txt target/` |
| `-v` | **verbose** — показывает подробности выполнения | `mv -v *.txt target/` |
| `-n` | **no-clobber** — не перезаписывать существующие файлы | `mv -n file.txt target/` |
| `-b` | **backup** — создаёт резервную копию перед перезаписью | `mv -b file.txt target/` |

---

## 💡 Комбинации и примеры

### 1. Переместить с подтверждением
```bash
mv -i document.txt /backup/
# Если файл существует, спросит: overwrite? (y/n)
```

### 2. Переместить только новые файлы
```bash
mv -u *.log /var/logs/archive/
# Переместит только те .log файлы, которых нет в archive или они новее
```

### 3. Переместить и показать что происходит
```bash
mv -v *.jpg images/
# Вывод: 'photo1.jpg' -> 'images/photo1.jpg'
```

### 4. Переместить все файлы определённого типа
```bash
mv *.pdf ~/documents/pdfs/
```

### 5. Переместить папку и переименовать одновременно
```bash
mv project/ /backup/old_project/
# Перемещает папку project в /backup/ и переименовывает в old_project
```

---

## ⚠️ Важные нюансы

### 🔴 Если назначение — папка
```bash
mv file.txt existing_folder/   # переместит ВНУТРЬ папки
```

### 🟡 Если назначение — не существующая папка
```bash
mv file.txt new_folder/        # если new_folder не существует — ошибка
mv file.txt new_folder         # если new_folder не существует — ПЕРЕИМЕНУЕТ файл в "new_folder"
```

### 🟢 Правило:
- Если **назначение существует и это папка** → файл перемещается внутрь
- Если **назначение не существует** → файл переименовывается
- Если **назначение существует и это файл** → файл перезаписывается (если нет `-i`)

---

## 🆚 Отличие `mv` от `cp`

| `mv` | `cp` |
|------|-----|
| Перемещает (стирает оригинал) | Копирует (сохраняет оригинал) |
| Работает мгновенно (просто меняет путь) | Копирует содержимое (занимает время) |
| Можно использовать для переименования | Нельзя переименовать без копирования |

---

## 🧠 Примеры для практики

```bash
# 1. Переименовать файл
mv report.txt final_report.txt

# 2. Переместить файл в другую папку
mv image.png ~/Pictures/

# 3. Переместить все .txt файлы в папку backup
mv *.txt backup/

# 4. Переместить с подтверждением и подробным выводом
mv -iv *.log /var/logs/

# 5. Переместить папку и всё её содержимое
mv old_project/ /home/user/archived_projects/

# 6. Переместить, но не перезаписывать существующие файлы
mv -n file.txt destination/

# 7. Создать резервную копию при перезаписи
mv -b config.conf /etc/config.conf
# Создаст /etc/config.conf~ (старую версию)
```

## 🎯 Короткая шпаргалка

| Задача | Команда |
|--------|---------|
| Переименовать файл | `mv старое_имя новое_имя` |
| Переименовать папку | `mv старая_папка новая_папка` |
| Переместить файл в папку | `mv файл папка/` |
| Переместить несколько файлов | `mv file1 file2 file3 папка/` |

| С подтверждением | `mv -i источник назначение` |

Используй `mv -iv` в комбинации:
- `-i` защитит от случайной перезаписи
- `-v` покажет, что именно произошло

Это особенно полезно при работе с важными файлами! 🚀

# 🔗 Ссылки в Linux — жёсткие и символические

## 🎯 Что такое ссылки?

Ссылки (links) в Linux — это способ создать **альтернативное имя** для файла. Они позволяют обращаться к одному и тому же содержимому из разных мест файловой системы.

В Linux существует **два типа ссылок**:
- **Жёсткие ссылки** (hard links)
- **Символические ссылки** (symbolic links, symlinks, "мягкие" ссылки)

---

## 📌 Жёсткие ссылки (Hard Links)

### Что это?
Жёсткая ссылка — это **дополнительное имя** для существующего файла. Она указывает непосредственно на **inode** файла (область на диске, где хранятся данные).

### Создание
```bash
ln исходный_файл имя_ссылки
```

### Пример
```bash
# Создаём файл
touch file.txt

# Создаём жёсткую ссылку
ln file.txt hard_link.txt

# Проверяем — это один и тот же файл
ls -li file.txt hard_link.txt
# 1234567 -rw-r--r-- 2 user user 0 Mar 22 10:00 file.txt
# 1234567 -rw-r--r-- 2 user user 0 Mar 22 10:00 hard_link.txt
#   ↑ один и тот же inode    ↑ счётчик ссылок = 2
```

### Особенности жёстких ссылок

| Свойство | Описание |
|----------|----------|
| ✅ **Делят один inode** | Несколько имён указывают на одни и те же данные |
| ✅ **Не занимают доп. место** | Это просто ещё одно имя для существующих данных |
| ✅ **Работают после удаления** | Пока есть хотя бы одна ссылка, данные сохраняются |
| ✅ **Счётчик ссылок** | `ls -l` показывает число в 2-й колонке |
| ❌ **Не работают с папками** | Нельзя создать жёсткую ссылку на директорию (кроме суперпользователя) |
| ❌ **Не работают между дисками** | Только в пределах одной файловой системы (раздела) |

### Когда использовать жёсткие ссылки?
- Для создания **резервных копий без дублирования** места
- Когда нужно, чтобы файл был доступен под разными именами
- Для экономии дискового пространства

---

## 🔗 Символические ссылки (Symbolic Links, Symlinks)

### Что это?
Символическая ссылка — это **специальный файл**, который содержит **путь к другому файлу или папке**. Работает как "ярлык" в Windows.

### Создание
```bash
ln -s исходный_файл имя_ссылки
```

### Пример
```bash
# Создаём файл
touch real_file.txt

# Создаём символическую ссылку
ln -s real_file.txt symlink.txt

# Проверяем
ls -l symlink.txt
# lrwxrwxrwx 1 user user 13 Mar 22 10:00 symlink.txt -> real_file.txt
# ↑ первая буква l — символьная ссылка   ↑ указывает на real_file.txt
```

### Особенности символических ссылок

| Свойство | Описание |
|----------|----------|
| ✅ **Могут ссылаться на папки** | Создавай ярлыки для любых директорий |
| ✅ **Работают между дисками** | Могут указывать на другую файловую систему |
| ✅ **Понятны в ls -l** | Видно, куда указывает → |
| ❌ **Занимают место** | Хранят путь (несколько байт) |
| ❌ **Ломаются** | Если удалить оригинал, ссылка становится "битой" (dangling) |
| ❌ **Свой inode** | У ссылки свой inode, не совпадает с оригиналом |

### Когда использовать символические ссылки?
- Для создания **ярлыков** на часто используемые папки
- Для **версионирования** (например, `current` → `app-1.2.3`)
- Для организации **структуры проекта**
- Когда нужно ссылаться на файлы на другом диске

---

## 📊 Сравнение жёстких и символических ссылок

| Характеристика | Жёсткая ссылка | Символическая ссылка |
|----------------|----------------|---------------------|
| **Команда** | `ln` | `ln -s` |
| **Работа с папками** | ❌ Нельзя | ✅ Можно |
| **Между разделами** | ❌ Нельзя | ✅ Можно |
| **Занимает место** | Нет (просто имя) | Да (хранит путь) |
| **После удаления оригинала** | Данные сохраняются | Ссылка битая |
| **Показывает в ls -l** | Обычный файл, число в 2 колонке | `l` в начале, стрелка → |
| **Свой inode** | Нет (общий с оригиналом) | Да |
| **Счётчик ссылок** | Увеличивается | Не влияет |

---

## 🔍 Полезные команды для работы со ссылками

### Найти все жёсткие ссылки на файл
```bash
# По номеру inode
find / -inum inode_номер 2>/dev/null

# Пример
ls -i file.txt          # узнаём inode
find / -inum 1234567    # ищем все ссылки
```

### Найти битые символические ссылки
```bash
find /path -type l -! -exec test -e {} \; -print
# или проще
find /path -xtype l
```

### Проверить, является ли файл ссылкой
```bash
test -L file.txt && echo "Символическая ссылка"
test -h file.txt && echo "Символическая ссылка"  # то же самое
```

### Показать, куда ведёт символическая ссылка
```bash
readlink symlink.txt
# выведет: real_file.txt

readlink -f symlink.txt   # полный путь (следует по ссылкам)
```

### Удалить ссылку
```bash
unlink symlink.txt
# или
rm symlink.txt
```

## 💡 Практические примеры

### 1. Символическая ссылка на папку
```bash
# Часто используемая папка в домашней директории
ln -s /var/www/html/project ~/project
# Теперь из ~/project можно быстро попасть в папку проекта
```

### 2. Управление версиями (versioning)
```bash
# После обновления приложения
ln -sfn app-2.0.0 current
# -s: символическая
# -f: перезаписать существующую
# -n: обращаться как с файлом (не входить в папку)

# Используем:
./current/bin/start.sh  # всегда последняя версия
```

### 3. Экономия места с жёсткими ссылками
```bash
# Вместо копирования больших файлов
cp large_file.iso backup/          # займёт 2× место
ln large_file.iso backup/          # займёт 1× место (жёсткая ссылка)
```

### 4. Создание нескольких имён для конфига
```bash
ln /etc/nginx/nginx.conf /etc/nginx/nginx.conf.backup
# Оба имени указывают на одни данные
# Можно менять через любое имя — изменения видны везде
```

### 5. Проверка битых ссылок после переноса
```bash
# Перенесли папку, ссылки сломались
find ~ -xtype l -ls   # покажет все битые ссылки
```

## ⚠️ Опасности и подводные камни

### 🟡 Удаление оригинала жёсткой ссылки
```bash
rm original.txt        # данные НЕ удалены, пока есть hard_link.txt
# Файл доступен через hard_link.txt
```

### 🔴 Удаление оригинала символической ссылки
```bash

```

### 🟡 Рекурсивное копирование с символическими ссылками

### 🟡 Циклические ссылки
```bash
ln -s folder1 folder2
ln -s folder2 folder1   # можно создать, но вызовет проблемы
# Осторожно! Может привести к бесконечным циклам в find
```

---

## 🧠 Примеры для практики

```bash
# 1. Создать жёсткую ссылку
ln file.txt hard.txt

# 2. Создать символическую ссылку на файл
ln -s file.txt soft.txt

# 3. Создать символическую ссылку на папку
ln -s /var/log ~/logs

# 4. Перезаписать существующую ссылку
ln -sf new_target old_link

# 5. Узнать, куда ведёт ссылка
readlink /usr/bin/python
# часто: python3

# 6. Найти все символические ссылки в текущей папке
find . -type l

# 7. Найти все жёсткие ссылки (счётчик > 1)
find . -type f -links +1
```

## 🎯 Короткая шпаргалка

| Задача | Команда |
|--------|---------|
| Создать жёсткую ссылку | `ln исходник ссылка` |
| Создать символьную ссылку | `ln -s исходник ссылка` |
| Перезаписать символьную ссылку | `ln -sf новый_исходник ссылка` |
| Посмотреть, куда ведёт ссылка | `readlink ссылка` |
| Найти битые ссылки | `find . -xtype l` |
| Удалить ссылку | `unlink ссылка` или `rm ссылка` |
| Узнать inode файла | `ls -i файл` |
| Проверить тип ссылки | `test -L файл && echo "symlink"` |

---

## 💡 Главное запомнить

1. **Жёсткая ссылка** = ещё одно имя для тех же данных (общий inode)
2. **Символическая ссылка** = файл-указатель (как ярлык)
3. **Жёсткие** работают только в пределах одного раздела и не работают с папками
4. **Символические** работают везде, но ломаются при удалении оригинала
5. При удалении файла данные исчезают, только когда **все жёсткие ссылки** удалены
6. Символические ссылки легко заметить в `ls -l` по букве `l` и стрелке `→`

Ссылки — мощный инструмент для организации файловой системы, экономят место и упрощают навигацию! 🚀
