Вот **расширенный конспект-таблица** содержимого основных каталогов в Linux (на примере современного Ubuntu/Debian с `usr-merged`). Данные приведены более подробно, включая подкаталоги и назначение.

| Каталог | Подробное содержимое и назначение |
| :--- | :--- |
| **`/bin`** | Символическая ссылка на `/usr/bin`. Исторически содержал основные пользовательские команды ( `ls`, `cp`, `mv`, `cat`, `echo`, `bash` ), необходимые даже в однопользовательском режиме. |
| **`/bin.usr-is-merged`** | Пустой служебный файл-метка для системы пакетов dpkg. Подтверждает, что слияние `/bin` → `/usr/bin` выполнено. |
| **`/boot`** | Статические файлы загрузчика: ядро (`vmlinuz-версия`), образ начальной RAM-диска (`initrd.img-версия`), конфигурация GRUB (`grub/`), карта символов (`System.map-версия`). |
| **`/cdrom`** | Точка монтирования оптического привода (CD/DVD/BD). Обычно пуста или содержит смонтированный диск. Используется некоторыми дистрибутивами как место автоматического монтирования. |
| **`/dev`** | Виртуальная ФС `devtmpfs`. Содержит файлы устройств, управляемые ядром: диски (`sda`, `sdb`), разделы (`sda1`, `sda2`), терминалы (`tty0`, `pts/0`), нулевое устройство (`null`), генератор случайных чисел (`random`, `urandom`), управление DVD (`sr0`). |
| **`/etc`** | Системные конфигурационные файлы (текстовые). Важные подкаталоги и файлы:<br>• `passwd`, `shadow`, `group` — учётные записи<br>• `fstab` — точки монтирования<br>• `hostname`, `hosts` — сетевые имена<br>• `network/` или `netplan/` — сеть<br>• `apt/` (Debian) или `yum.repos.d/` (RHEL) — пакетный менеджер<br>• `systemd/` — настройки systemd<br>• `ssh/` — ключи и конфиги SSH<br>• `cron.d/`, `crontab` — задачи по расписанию |
| **`/home`** | Личные домашние каталоги обычных пользователей (не root). Структура: `/home/username/` с подпапками `Desktop/`, `Documents/`, `Downloads/`, `.config/` (скрытые настройки программ). |
| **`/lib`** | Ссылка на `/usr/lib`. Раньше содержал общие библиотеки (`.so`), необходимые для исполняемых файлов из `/bin` и `/sbin`, а также модули ядра (`modules/`). |
| **`/lib64`** | Ссылка на `/usr/lib64` (в некоторых дистрибутивах). Содержит 64-битные библиотеки (например, `ld-linux-x86-64.so.2`). |
| **`/lib.usr-is-merged`** | Аналогичная метка слияния для `/lib` → `/usr/lib`. |
| **`/lost+found`** | Создаётся только для файловых систем ext3/ext4. При восстановлении после сбоя (`fsck`) утерянные фрагменты (файлы без имён) помещаются сюда. Обычно пуст. |
| **`/media`** | Точки автоматического монтирования съёмных носителей: USB-флешки, внешние диски, карты памяти. Подкаталоги: `/media/username/метка_устройства/`. |
| **`/mnt`** | Ручная точка монтирования — для временного подключения файловых систем администратором (например, `mount /dev/sdb1 /mnt`). |
| **`/opt`** | Дополнительное программное обеспечение, не входящее в состав дистрибутива (адд-оны, коммерческие программы). Каждая программа — в своей подпапке: `/opt/google/chrome/`, `/opt/teamviewer/`. Исполняемые файлы часто через симлинки в `/usr/bin`. |
| **`/proc`** | Виртуальная псевдо-ФС `procfs` (в оперативной памяти). Информация о процессах и ядре:<br>• `cpuinfo`, `meminfo`, `uptime` — системные параметры<br>• `PID/` — каталог каждого процесса (environ, fd, status)<br>• `self/` — ссылка на текущий процесс<br>• `mounts`, `partitions` — смонтированные ФС |
| **`/root`** | Домашний каталог суперпользователя root. Имеет права доступа 700 (только для root). Хранит личные файлы, скрипты, конфиги (`.bashrc`), историю команд (`.bash_history`). |
| **`/run`** | Виртуальная ФС `tmpfs`. Динамические данные с момента загрузки:<br>• `user/1000/` — runtime данные пользователя<br>• `systemd/` — сокеты, журналы<br>• `lock/` — файлы блокировок<br>• `mount/` — временные точки монтирования |
| **`/sbin`** | Ссылка на `/usr/sbin`. Раньше — системные команды для администрирования (`fdisk`, `mkfs`, `ifconfig`, `reboot`, `iptables`). |
| **`/sbin.usr-is-merged`** | Метка слияния `/sbin` → `/usr/sbin`. |
| **`/snap`** | Каталог для пакетов Snap (приложения в песочнице от Canonical). Содержит смонтированные squashfs-образы snap-пакетов и их данные. Используется в Ubuntu и производных. |
| **`/srv`** | Данные сервисов, предоставляемых системой (служб):<br>• `www/` — файлы веб-сервера (Apache, Nginx)<br>• `ftp/` — FTP-репозиторий<br>• `git/`, `svn/` — репозитории контроля версий<br>• `cvs/` |
| **`/swap.img`** | Файл подкачки (swap), если используется вместо или вместе с swap-разделом. Выступает в роли виртуальной памяти — расширяет RAM на диск. |
| **`/sys`** | Виртуальная ФС `sysfs` (связана с ядром). Информация об устройствах, драйверах, шинах, питании, иерархии устройств:<br>• `class/` — классы устройств (net, input, sound)<br>• `block/` — блочные устройства<br>• `power/` — управление питанием<br>• `firmware/` — прошивки |
| **`/tmp`** | Временные файлы (очищается при перезагрузке, права 1777 — sticky bit). Используется программами для кэша, сессионных данных. Может быть смонтирован как `tmpfs` (в оперативной памяти). |
| **`/usr`** | **Unix System Resources** — вторая основная иерархия. Важные подкаталоги:<br>• `bin/`, `sbin/`, `lib/` — основные исполняемые файлы, библиотеки<br>• `local/` — программы, собранные вручную (аналог /usr, но не перезаписывается при обновлении системы)<br>• `share/` — архитектурно-независимые данные (man, doc, icons, locale)<br>• `include/` — заголовочные файлы C/C++<br>• `src/` — исходные коды (редко) |
| **`/var`** | **Variable** — часто изменяемые данные во время работы системы:<br>• `log/` — системные и прикладные логи (syslog, auth.log, dmesg)<br>• `cache/` — кэши пакетного менеджера (apt, pacman)<br>• `spool/` — очереди заданий (печать, почта)<br>• `tmp/` — долгоживущие временные файлы<br>• `lib/` — динамические данные служб (БД, Docker, MySQL)<br>• `mail/` — почтовые ящики пользователей |

### Дополнительные примечания:

1. **usr-merged** — в современных Linux большинство корневых каталогов (`/bin`, `/sbin`, `/lib`) являются символическими ссылками на соответствующие подкаталоги в `/usr`. Это упрощает управление.

2. **Символические ссылки** — в таблице указано, что некоторые папки — это ссылки (стрелочки), но физически данные хранятся в `/usr`.

3. **Права доступа** — многие папки требуют прав root для записи (`/etc`, `/boot`, `/root`), а `/tmp` имеет sticky bit.

4. **Отличия дистрибутивов**:
   - В **Red Hat / Fedora** нет папок `.usr-is-merged` и `/snap`, но может быть `/mnt/cdrom` вместо `/cdrom`.
   - В **Arch Linux** тоже нет меток слияния, но структура аналогична.
   - В **openSUSE** есть `/run/media/` для автоматического монтирования вместо `/media`.

5. **Точки монтирования** — `/proc`, `/sys`, `/dev`, `/run` — это виртуальные ФС, которые создаются в ОЗУ при загрузке.
