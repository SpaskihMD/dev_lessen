# 🐧 Монтирование файловых систем в Linux

> 📌 **Кратко:** Монтирование — это процесс подключения устройства (диска, флешки, ISO) к папке, после чего содержимое устройства становится доступным в этой папке.

## 🔧 Основные команды

### 👁️ Просмотр смонтированных систем

```bash
mount                     # Список всех смонтированных систем
findmnt                   # Древовидный вывод монтирования
lsblk -f                  # Устройства и их файловые системы
df -h                     # Занятое место на смонтированных дисках
```

### 🔌 Монтирование (`mount`)

**Синтаксис:**
```bash
sudo mount [-t тип_фс] [-o опции] устройство точка_монтирования
```

**Примеры:**

| Действие | Команда |
|----------|---------|
| Автоопределение типа | `sudo mount /dev/sdb1 /mnt/usb` |
| Явное указание типа ext4 | `sudo mount -t ext4 /dev/sda2 /mnt/data` |
| Только чтение | `sudo mount -o ro /dev/sdc1 /mnt/disk` |
| Монтирование ISO-образа | `sudo mount -o loop ~/image.iso /mnt/iso` |

### 🎛️ Полезные опции (`-o`)

| Опция | Значение |
|:------|:---------|
| `ro` | 🔒 Только чтение |
| `rw` | 📝 Чтение и запись |
| `noexec` | 🚫 Запрет выполнения программ |
| `nosuid` | 🔐 Игнорирование бит suid/sgid |
| `noatime` | ⚡ Не обновлять время доступа (ускоряет работу) |
| `umask=000` | 🔓 Полные права доступа (rwxrwxrwx) |
| `utf8` | 🌍 Поддержка русских имён файлов |

### ⛔ Отмонтирование (`umount`)

```bash
sudo umount /mnt/usb          # Отмонтировать по точке монтирования
sudo umount /dev/sdb1         # Отмонтировать по устройству
sudo umount -l /mnt/usb       # Ленивое отмонтирование (lazy)
sudo umount -f /mnt/usb       # Принудительное отмонтирование
```

## 💡 Практические примеры

### 💽 USB-флешка (FAT32)

```bash
sudo mkdir -p /media/usb
sudo mount /dev/sdb1 /media/usb
# С поддержкой русского языка
sudo mount -t vfat /dev/sdb1 /media/usb -o utf8,umask=000
```

### 🪟 NTFS (Windows)

```bash
# Установка драйвера (если нет)
sudo apt install ntfs-3g

sudo mount -t ntfs-3g /dev/sdb1 /media/ntfs -o utf8,umask=000
```

### 🗂️ ISO-образ

```bash
sudo mount -o loop ~/Downloads/ubuntu.iso /mnt/iso
ls /mnt/iso   # Просмотр содержимого образа
```

### 🌐 Сетевая папка NFS

```bash
sudo mount -t nfs 192.168.1.100:/srv/share /mnt/nfs
```

### 🖥️ SMB/CIFS (Windows-шара)

```bash
sudo mount -t cifs //192.168.1.100/share /mnt/smb -o username=user,password=pass
```

## ⚙️ Автоматическое монтирование (fstab)

Файл **`/etc/fstab`** содержит правила автоматического монтирования при загрузке системы.

### 📝 Формат записи

```
[устройство]  [точка_монтирования]  [тип_ФС]  [опции]  [dump]  [pass]
```

### ✅ Примеры записей

```bash
# Локальный раздел Linux
/dev/sda2   /mnt/data   ext4   defaults        0   2

# По UUID (рекомендуется, не зависит от имени диска)
UUID=12345678-9abc-def0-1234-567890abcdef   /mnt/data   ext4   defaults   0   2

# EFI раздел
UUID=ABCD-1234   /boot/efi   vfat   defaults   0   1

# Сетевая шара с файлом паролей
//server/share   /mnt/share   cifs   credentials=/etc/smbcreds,uid=1000   0   0
```

### 🔄 Применение изменений fstab

```bash
sudo mount -a              # Смонтировать всё из fstab
sudo mount -a -v           # Подробный вывод
```

> ⚠️ **Важно:** После редактирования fstab не нужно перезагружаться — достаточно `sudo mount -a`. Но перед этим убедитесь, что точки монтирования существуют (`mkdir -p`).

## 🚨 Ошибки и их решение

| Ошибка | Решение |
|:-------|:--------|
| `mount point does not exist` | `sudo mkdir -p /путь/к/точке` |
| `device is busy` | `lsof /точка` или `fuser -k /точка` |
| `unknown filesystem type` | Установить драйвер: `ntfs-3g`, `exfat-fuse` |
| `only root can mount` | Использовать `sudo` или добавить `user` в опции fstab |
| `Structure needs cleaning` | Запустить `fsck` для восстановления ФС |

### 🔍 Как найти процесс, блокирующий отмонтирование

```bash
lsof /mnt/usb           # Список процессов, использующих точку
fuser -v /mnt/usb       # Альтернативный вариант
fuser -km /mnt/usb      # Убить все процессы, мешающие отмонтированию
```

## 👤 Монтирование без sudo

### 🆕 udisksctl (современный способ, работает в большинстве дистрибутивов)

```bash
udisksctl mount -b /dev/sdb1       # Смонтировать
udisksctl unmount -b /dev/sdb1     # Отмонтировать
udisksctl power-off -b /dev/sdb1   # Безопасно отключить устройство
```

### 📦 pmount (легковесная альтернатива)

```bash
sudo apt install pmount            # Установка

pmount /dev/sdb1 mydisk            # Смонтирует в /media/mydisk
pumount /dev/sdb1                  # Отмонтировать
```

## ⚡ Шпаргалка быстрых команд

```bash
# 📋 Посмотреть все диски и разделы
lsblk

# 🏷️ Узнать тип файловой системы и UUID
blkid /dev/sdb1
lsblk -f /dev/sdb1

# 🔌 Смонтировать вручную
sudo mount /dev/sdb1 /mnt/usb

# ⛔ Отмонтировать
sudo umount /mnt/usb

# 🔄 Перемонтировать в режим чтения-записи
sudo mount -o remount,rw /mnt/data

# 📦 Смонтировать всё из fstab
sudo mount -a

# 🧹 Найти и убить процессы, блокирующие отмонтирование
fuser -km /mnt/usb && sudo umount /mnt/usb
```
